## 摘要
我们介绍了ABot-OCR，一种端到端的视觉语言模型，能够在单次前向传递中直接将页面图像转录为干净的Markdown。通过这种方式，我们的方法完全消除了脆弱的模块化编排需求。为了最大化解析保真度，我们开发了一个专门的数据引擎，以提供大规模、结构一致的监督。此外，我们提出了解耦异构文档优化，一种结构约束的强化学习方法，它提高了文本准确性，并严格确保了标记格式的良好形式，超越了单一的监督微调。大量评估证明了我们框架的优越性能。在OmniDocBench v1.5和v1.6基准测试中，ABot-OCR在所有端到端系统中达到了92.81和93.30的最先进分数，显著缩小了与强大流程基线的性能差距。最后，涵盖十种不同语言的全面多语言文本识别进一步证实了ABot-OCR的稳健泛化能力。

## Abstract
We introduce ABot-OCR, an end-to-end vision-language model that transcribes a page image directly into clean Markdown in a single forward pass. By doing so, our approach completely eliminates the need for brittle modular orchestration. To maximize parsing fidelity, we develop a dedicated data engine to provide large-scale, structurally consistent supervision. Furthermore, we propose Decoupled Heterogeneous Document Optimization, a structure-constrained reinforcement learning method that sharpens textual accuracy and strictly enforces markup well-formedness beyond supervised fine-tuning alone. Extensive evaluations demonstrate the superior performance of our framework. On the OmniDocBench v1.5 and v1.6 benchmarks, ABot-OCR achieves state-of-the-art scores of 92.81 and 93.30 among all end-to-end systems, substantially narrowing the performance gap relative to strong pipeline baselines. Finally, comprehensive multilingual text recognition across ten diverse languages further confirms the robust generalizability of ABot-OCR.

- **核心问题**：文档解析（Document Parsing）旨在将视觉无结构的文档图像转化为文本结构化的格式（如 Markdown）。传统方法多采用多模块流水线（pipeline）方式，将任务分解为布局分析、文本识别、公式识别、表格识别等子任务，流程脆弱、误差累积、部署复杂。近年来，基于视觉语言模型（VLM）的端到端方法兴起，但性能仍落后于流水线系统，尤其在高保真结构解析（如表格、公式、阅读顺序）上存在明显不足。
- **整体含义**：本文提出 ABot-OCR，一个 2B 参数的端到端 VLM，旨在通过单次前向传递直接将文档图像转录为干净 Markdown，消除模块化编排，在保持高识别精度的同时严格保证结构正确性，从而缩小端到端模型与传统流水线方法之间的性能差距，推动文档解析向统一、简洁、高性能的方向发展。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **统一端到端框架**：以 Qwen3-VL-2B-Instruct 为骨干，将文档解析视为直接图像到 Markdown 的生成任务。
- **三级渐进训练策略**：分为模块化基础训练、端到端统一训练、结构约束强化学习（DHDO）三个阶段。
- **专用数据引擎**：通过层次一致性注释验证和网络规模文档伪标签生成高质量、大规模训练数据。

### 关键技术细节

#### 数据引擎（Data Engine）
- **层次一致性注释验证**（Hierarchical-Consistency Annotation Verification）：
  - 第一层（语言一致性）：不依赖图像，检查注释格式正确性（如 LaTeX 括号平衡、HTML 表格标签嵌套合法、文本标注框坐标合法）。
  - 第二层（视觉一致性）：使用布局分析 + 专业识别模型（如公式、表格识别器）对区域进行标注，与原始注释进行一致性打分（布局对齐、文本相似度、表格结构等价等）。
  - 第三层（VLM 辅助评分）：引入大参数多模态推理模型，计算文档解析一致性分数（DPCS），包含文本保真度、布局定位、阅读顺序、结构保真度、格式有效性、语义完整性六个维度加权和。
  - 统一质量控制：DPCS ≥ 80 为高置信数据；60-80 加权保留；<60 的样本重新标注或过滤。
- **网络规模文档伪标签**（Web-Scale Document Pseudo-Labeling）：
  - 对爬取的网络文档，通过布局分析分割区域，用专业模型标注区域内容（公式、表格、文本等），按阅读顺序组装成页面级结构化表示。
  - 所有伪标签样本同样经过 DPCS 门控。

#### 三级训练策略

**Stage 1：模块化文档解析（Modular Document Parsing）**
- 分解为四个子任务：文本定位（Text Spotting）、公式识别（Formula Recognition）、表格识别（Table Recognition）、布局分析（Layout Analysis）。
- 每个子任务有特定 prompt 和输出格式，通过交叉熵损失优化。
- 数据规模约 1000 万样本。

**Stage 2：端到端文档解析（End-to-End Document Parsing）**
- 使用页面级 Markdown 标注，进行监督微调。
- 严格规范化 Markdown 格式：显示公式用 `$$...$$`，行内公式用 `$...$`，表格用 HTML，阅读顺序遵循视觉布局，标题层级不跳级。
- 数据规模约 140 万样本。

**Stage 3：结构约束强化学习（Decoupled Heterogeneous Document Optimization, DHDO）**
- **奖励设计**：
  - 感知奖励（Content Accuracy）：基于 Levenshtein 相似度，去除不影响视觉一致性的空白后计算。
  - 结构奖励包括三项：
    1. 公式语法正确性（Formula Syntactic Soundness）：KaTeX 编译检查 + CDM（Character Detection Matching）视觉等价性。
    2. 表格结构有效性（Table Structural Validity）：HTML 标签嵌套合法、行宽一致、行数对齐，并结合树编辑距离（TEDS）。
    3. 一般结构闭合性（General Structural Closure）：Markdown 定界符和 HTML 表格标签配对平衡，标题层级有效，列表缩进正确。
  - **感知条件化**：结构奖励仅在感知奖励超过阈值（0.7）时才激活，避免奖励欺骗。
- **奖励聚合与策略优化**：
  - 对每个奖励维度先进行组内标准化（group-wise normalization），再加权求和，最后进行 batch 级再缩放。
  - 聚合权重：`w_text : w_formula : w_table : w_struct = 1.0 : 0.8 : 0.8 : 0.5`。
  - 使用 PPO 的裁剪代理目标，并加入 KL 散度惩罚锚定 Stage 2 checkpoint。
- **对比 GRPO**：DHDO 避免了 GRPO 多奖励归一化导致的优势坍缩（advantage collapse），保留细粒度奖励信号。

#### 其他细节
- 骨干模型：Qwen3-VL-2B-Instruct。
- 训练超参数：前两阶段学习率 5e-5，全局批次 128；第三阶段学习率 5e-7，使用核采样（p=0.99, k=50）。

## 6. 主要结论与发现

- ABot-OCR 在 OmniDocBench v1.5 和 v1.6 上分别取得 **92.81** 和 **93.30** 的 Overall 分数，**均达到端到端系统 SOTA**，显著缩小了与最高流水线系统的差距（v1.5 上比最优流水线 PaddleOCR-VL-1.5 94.50 低约 1.7 分，v1.6 上比最优流水线 MinerU2.5-Pro 95.69 低约 2.4 分）。

## 7. 优点

- **端到端简洁性**：单模型、单次前向完成全页解析，避免复杂流水线和误差累积。
