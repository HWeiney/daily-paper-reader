## 摘要
多模态大语言模型通常以端到端方式训练以预测真实答案，但监督信号仅应用于文本标记。视觉标记作为视觉信息的核心载体，仅作为上下文的一部分隐式优化，导致粗粒度的视觉理解。先前的工作试图监督视觉输入，但不可避免地依赖辅助组件（如额外的解码器或前向传播），因为视觉标记缺乏易于解释的标签。这限制了其实用性。在这项工作中，我们提出了直接视觉监督微调（DV-SFT），它为视觉标记构建显式的标记级监督，并通过与文本相同的下一个标记预测目标来训练它们。具体来说，我们利用OCR相关场景中的直接视觉-文本对应关系，自动为每个视觉标记标注其对应图像块中的单词。DV-SFT将MLLM视为黑盒，无需架构修改或额外前向传播。大量实验证明了直接视觉监督的优越性。DV-SFT在三个域内和四个域外基准测试中始终优于标准SFT。进一步分析表明，视觉监督有效增强了细粒度视觉理解，并实现了更高的多模态对齐效率。

## Abstract
Multimodal large language models are typically trained end-to-end to predict ground-truth answers, yet supervision signals are applied exclusively to text tokens. Visual tokens, the core carriers of visual information, are optimized only implicitly as part of the context, leading to coarse-grained visual understanding. Prior works attempt to supervise visual inputs but inevitably rely on auxiliary components such as additional decoders or forward passes, because visual tokens lack readily interpretable labels. This limits their practical applicability. In this work, we propose \textbf{D}irect \textbf{V}ision \textbf{S}upervised \textbf{F}ine-\textbf{T}uning (DV-SFT), which constructs explicit, token-level supervision for visual tokens and trains them through the same next-token prediction objective used for text. Specifically, we exploit the direct vision--text correspondence in OCR-related scenarios and automatically label each visual token with the word in its corresponding image patch. DV-SFT treats the MLLM as a black box, requiring no architectural modifications or additional forward passes. Extensive experiments demonstrate the superiority of direct vision supervision. DV-SFT consistently outperforms standard SFT across three in-domain and four out-of-domain benchmarks. Further analyses show that vision supervision effectively enhances fine-grained visual understanding and achieves higher multimodal alignment efficiency.

## 1. 核心问题与整体含义（研究动机和背景）

- **现有问题**：多模态大语言模型（MLLM）采用端到端训练，但监督信号**仅作用于文本 token**，视觉 token 作为视觉信息的主要载体只能通过上下文**隐式优化**，导致对细粒度视觉细节的理解粗放，容易产生幻觉（hallucination）。
- **前期工作局限**：部分研究尝试对视觉 token 施加监督，但通常需要额外解码器、前向传播或架构修改，且监督信号间接（如奖励信号、表示对齐、像素重建、下一视觉 token 预测），与视觉理解目标不一致。
- **论文动机**：作者发现 MLLM 中视觉 token 的 logits 分布已自发包含部分语义信息（如对应图像补丁中的词语），**直接利用这些自然对齐信息作为显式标签**，有望在不增加模型复杂度的情况下提升细粒度视觉理解。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **DV-SFT**：利用 OCR 场景中**直接的视觉-文本对应关系**，为每个视觉 token 自动标注其对应图像补丁中的单词，然后通过**标准的 next-token prediction 损失**同时训练视觉 token 和文本 token。
- **无需修改模型架构、无需额外前向传播**，将 MLLM 视为黑盒。

### 2.2 视觉标签构建（Vision Label Construction）
- **Image-to-Label**：使用文本检测+识别模型（PaddleOCR）提取图像中所有单词及其空间位置，将每个单词对齐到其**右下角对应的视觉 token**，过滤掉超大词或分词长度超过3的词，避免同一 token 被多个标签覆盖。
- **Label-to-Image**：针对纯文本问答数据集（如 SQuAD），通过绘图工具将单词按 patch 网格精确渲染到空白图像上，保证每个单词只监督其**最后一个视觉 token**（符合自回归性质）。

### 2.3 损失函数
- **文本监督损失**（标准 SFT）：  
  \( L_t = -\frac{1}{n} \sum \log P(y_i | I, X, Y_{<i}; \theta) \)
- **视觉监督损失**：  
  \( L_v = -\frac{1}{m} \sum \log P(w_i | I_{\le i}; \theta) \)  
  其中 \( w_i \) 是第 i 个视觉 token 对应的单词标签。
- **Vision Smoothing**：考虑到视觉编码器的双向特性导致信息跨 token 流动，引入类似标签平滑的机制，让每个视觉 token 不仅学习自身标签，还学习图像中其他视觉标签的分布：  
  \( L_v = -\frac{1}{m} \sum_{i=1}^m \left[ (1-\beta) \log P(w_i | I_{\le i}) + \frac{\beta}{|V|-1} \sum_{t \neq w_i} \log P(t | I_{\le i}) \right] \)
- **最终损失**：\( L = L_t + \lambda L_v \)，其中 \(\lambda\) 控制视觉监督权重。

## 6. 主要结论与发现

- **直接视觉监督显著提升细粒度视觉理解**：DV-SFT 在全部域内和域外基准上一致优于标准 SFT，**仅在 2B 模型上平均提升 0.5~0.9 个百分点，8B 模型上平均提升 0.5~1.0 个百分点**。
- **多模态对齐效率更高**：训练早期（尤其是 out-of-domain）DV-SFT 的优势更明显，说明直接监督视觉 token 能更快、更稳定地建立视觉语义。
- **字符识别能力增强**：在 Non-Contextual OCR 任务上提升更大，说明模型学到了**更纯粹的视觉特征**而非单纯依赖语言先验。
- **通用视觉能力不退化**：在 MME 基准上，DV-SFT 的 Accuracy/F1 略高于 SFT，证明**基于 OCR 场景的单词监督可以迁移到非文字场景**，且不会损害模型的一般视觉能力。
- **Vision Smoothing 与损失平衡至关重要**：过大或过小的视觉监督权重都会导致性能下降；Vision Smoothing 有效缓解了视觉编码器双向注意力带来的跨 token 信息干扰。

## 7. 优点

- **简洁高效**：无需修改模型架构、无需额外前向传播或解码器，完全兼容现有 MLLM 训练流程，即插即用。
- **黑盒适用**：仅需训练数据中的图像和文本，不依赖模型内部中间状态，可应用于任意 MLLM。
- **可解释性强**：视觉标签直接来源于图像中的单词，监督信号直观、可控。
- **鲁棒性好**：在 2B/8B/1.7B 三种规模模型上均取得一致提升，且泛化到多个域外任务。

## 8. 不足与局限

- **局限于 OCR 场景**：视觉标签的构建依赖文本-视觉的精确对应，仅适用于图片中**包含可识别文字**的场景。对于纯物体、场景等无文字图像，无法直接应用（论文将其列为未来工作）。
- **单标签对齐假设**：一个视觉 token 在训练时只对应一个单词标签，但实际情况中一个 patch 可能包含多个语义（如多个字符或物体），单标签可能导致信息损失。
- **覆盖不完全**：训练数据中只有部分视觉 token 被分配了标签（DocVQA 覆盖率约7%，InfographicVQA 约12%，SQuAD 约31%），未覆盖的 token 仍只能通过文本损失间接学习。
- **未公开代码与超参数敏感性**：论文声称代码将开源，但实验部分仅给出推荐超参数 \(\beta=0.3, \lambda=2e-3\)，未系统讨论超参数敏感度。
- **算力报告不够详细**：仅提到使用 2 张 A100，未给出每个实验的具体运行时间，不利于复现与成本估算。
